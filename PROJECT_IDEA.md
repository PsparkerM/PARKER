# P.A.R.K.E.R.
## Personal Adaptive Resource & Kinetic Energy Regulator
### Полная проектная спецификация

**Торговый знак:** P.A.R.K.E.R.®
**Текущий этап:** Закрытое MVP на 3 пользователей
**Следующий этап:** Открытый доступ + масштабирование + AI-автоматизация

---

## 1. МИССИЯ ПРОЕКТА

Создать систему, которая работает как **личный нутрициолог + тренер + биохакер**
в кармане у каждого человека — вне зависимости от его графика, физических ограничений
и уровня подготовки. Не «универсальный план», а **живой адаптивный протокол**,
который перестраивается под реальную жизнь пользователя каждый день.

---

## 2. ПРОБЛЕМА

Проблема не в отсутствии информации — в её **неадаптивности к реальной жизни**.

### 2.1 Конфликт графика и режима
Стандартные планы требуют 5 приёмов пищи и 8 часов сна.
Для Петра (16–19ч работы) или Леры (12ч на ногах) — это физически невозможно.
Результат: человек срывается на 3-й день и бросает.

### 2.2 Игнорирование состояния здоровья
Большинство ботов и приложений дают «базовую программу» без учёта здоровья.
Для Вики со сколиозом 2-й степени стандартная тренировочная программа —
прямой путь к травме и ухудшению осанки.

### 2.3 Отсутствие гибкости при срывах
Если пользователь пропустил приём пищи или тренировку —
приложения «штрафуют» прогресс, вызывают чувство вины и заброс.
P.A.R.K.E.R. предлагает «Минимальный план» как страховку.

### 2.4 Метаболический хаос при нестандартных параметрах
Для Петра (рост 205 см, вес 75 кг) стандартные калькуляторы КБЖУ
ошибаются на 500–1000 ккал. Прогресс стоит на месте несмотря на усилия.

### 2.5 Разрыв между питанием и тренировками
Приложения для питания не знают о тренировках.
Приложения для тренировок не знают о диете и дефиците сна.
Никто не смотрит на человека целиком.

---

## 3. РЕШЕНИЕ — ПРИНЦИП «БИОЛОГИЧЕСКОГО АУДИТА»

P.A.R.K.E.R. не выдаёт статичный план — он строит **живой протокол**,
который адаптируется под вводные данные каждого дня.

### 3.1 Адаптивное питание
- Для Петра: «жидкие калории» + высококалорийные перекусы за 2 мин в перерыве 19-часовой смены
- Для Вики: 4–5 сбалансированных приёмов с учётом 10-часового рабочего дня
- Для Леры: антиотёчный протокол, контроль натрия, микропериодизация дефицита

### 3.2 Корректирующая активность
- Для Вики: исключение осевых нагрузок, ЛФК-блоки для осанки в структуре тренировки
- Для Леры: чередование зала / плавания / домашних тренировок по нагрузке на ноги
- Для Петра: минимальный тренировочный объём с максимальным анаболическим стимулом

### 3.3 Управление восстановлением
- Если пользователь отработал 12ч+ на ногах → силовая заменяется на плавание или растяжку
- Нестабильный сон → пониженная интенсивность тренировок + акцент на восстановление
- Сигналы стресса → снижение объёма нагрузки, витаминный блок

### 3.4 Обратная связь (этап 3+)
- Ежедневные отчёты: вес, самочувствие, сон, уровень стресса
- Анализ динамики: графики прогресса, адаптация плана
- AI-корректировка: модель перестраивает план на основе данных пользователя

## 4. ЦЕЛЕВАЯ АУДИТОРИЯ

### 4.1 MVP (закрытый, 3 человека)
Пётр, Вика, Валерия — реальные пользователи с известными параметрами.
Цель: отшлифовать алгоритм на живых кейсах перед открытием.

### 4.2 Основная аудитория (после открытия)

**«High-Load Профессионалы»**
Люди с графиком 12+ часов: HoReCa, медицина, IT, логистика, охрана.
Нет времени и сил на стандартный фитнес. Нужно максимум результата при минимуме усилий.

**«Специфические Атлеты»**
Нестандартные параметры (очень высокий / низкий рост, экстремальный вес)
или ограничения здоровья (сколиоз, проблемы с коленями, гипертония).

**«Результатники-эстеты»**
Хотят конкретный визуальный результат: рельеф, объёмы, пропорции.
Готовы считать КБЖУ и следовать микропериодизации.

### 4.3 Масштабирование (этап 4–5)
- Массовый рынок через Mini App с подпиской
- B2B: корпоративное здоровье для компаний с посменным графиком
- White-label решение для фитнес-клубов и диетологов

## 7. КОНЦЕПЦИЯ — ЦИФРОВОЙ ДВОЙНИК ЭКСПЕРТА

P.A.R.K.E.R. — это не просто бот. Это **цифровой двойник живого тренера**.
Чтобы заменить специалиста, система обладает тремя характеристиками:
**глубокий анализ данных → гибкость алгоритмов → контроль в реальном времени.**

Человек просто выполняет команды — P.A.R.K.E.R. берёт на себя всю математику,
биохимию и планирование. Второй пилот.

## 8. БЛОК «БИОМЕТРИЧЕСКИЙ ПРОФИЛЬ» (Входящие данные)

Чтобы система не ошибалась (как в случае с Петром 205 см),
P.A.R.K.E.R. собирает «цифровой слепок» пользователя.

### 8.1 Базовые параметры
- Рост, вес, возраст, пол

### 8.2 Антропометрия (замеры)
| Замер | Зачем |
|---|---|
| Обхват шеи | Точность формул (ВМФ-метод для % жира) |
| Обхват груди | Контроль массонабора у Петра |
| Обхват талии (уровень пупка) | Главный маркер жировой ткани |
| Обхват бёдер | Контроль у Леры |
| Обхват бицепса | Прогресс мышечной массы |
| Обхват бедра (ляшки) | Контроль у Леры |

> Вес может стоять, а объёмы уходить (как у Леры). Бот должен это понимать
> и не паниковать, если весы не двигаются.

### 8.3 Анализ образа жизни
- Рабочий график (часы стоя / сидя)
- Среднее количество сна (часов)
- Уровень бытовой активности (шаги в день)
- Тип активности: зал / дом / бассейн / на улице

### 8.4 Медицинские ограничения
- Травмы и хронические заболевания (сколиоз, колени, давление)
- Аллергии и непереносимость (лактоза, глютен, орехи)
- Пищевые предпочтения и антипатии (не люблю рыбу, обожаю креветки)

## 9. МОДУЛЬ «АДАПТИВНОЕ ПИТАНИЕ»

P.A.R.K.E.R. не выдаёт статичное меню. Он строит **динамический рацион**.

### 9.1 Персонализация выбора
**Система исключений (Blacklist):**
Если пользователь ставит бан на рыбу → алгоритм мгновенно пересчитывает
источники Омега-3 и белка через морепродукты (креветки) или растительные аналоги.

**Конструктор приёмов пищи:**
Бот выдаёт не «рецепт из интернета», а баланс:
> «Тебе нужно 40г белка и 60г углеводов. Выбери: Курица+Рис или Креветки+Паста»

### 9.2 Контроль напитков
- **Водный баланс:** расчёт нормы воды с учётом кофеина (кофе выводит воду) и тренировок
- **Жидкие калории:** для Петра (19ч работы) — смузи и гейнеры, которые можно выпить на ходу

### 9.3 Дневник питания в один клик
Пользователь пишет: `«съел 200г творога»`
→ P.A.R.K.E.R. сам вносит в базу, пересчитывает остаток КБЖУ на день.

### 9.4 Алгоритм адаптации
| Ситуация | Действие системы |
|---|---|
| Пропущен приём пищи | Перераспределяет КБЖУ на оставшиеся приёмы |
| Нет рыбы в рационе | Заменяет на морепродукты или растительный Омега-3 |
| График 16ч+ | Переходит в режим «жидких калорий» |
| Нестабильный сон | Добавляет магний, снижает кофеин во второй половине дня |
| Нет прогресса 2 недели | Пересчитывает КБЖУ автоматически |

## 10. МОДУЛЬ «ИНТЕЛЛЕКТУАЛЬНЫЙ ТРЕНИНГ»

Бот понимает **где находится пользователь** и **сколько у него сил**.

### 10.1 Гибридный подход
Переключение одной кнопкой:
- «Сегодня я **в зале**» → программа для зала
- «Сегодня я **дома**» → программа без оборудования
- «Сегодня я **в бассейне**» → плавательный протокол
- «Сегодня я **разбит**» → восстановительная растяжка / ЛФК

### 10.2 Техника выполнения
- Для каждого упражнения — короткое видео или гифка (этап 4)
- Для Вики: система **подсвечивает** упражнения для осанки
  и **блокирует** осевые нагрузки на позвоночник
- Текстовые подсказки по технике в MVP

### 10.3 Прогрессия нагрузок
P.A.R.K.E.R. помнит историю:
> «На прошлой неделе ты присел 40 кг × 8 повторений.
> Сегодня: 42.5 кг или 40 кг × 10 повторений.»

### 10.4 Коррекция по состоянию
| Состояние | Действие |
|---|---|
| Сон < 5ч | «Снижаем интенсивность на 30%. Базовая тренировка.» |
| 12ч на ногах (Лера) | Заменить ноги → верх тела или плавание |
| Высокий стресс | Снизить объём, добавить дыхательные практики |
| Отлично себя чувствую | «Сегодня можно добавить рабочий вес» |

## 11. СИСТЕМА КОНТРОЛЯ И ОБРАТНОЙ СВЯЗИ

### 11.1 Ежедневный чек-ап
Бот присылает утром:
- «Как спал?» (шкала 1–5 или часы)
- «Сколько воды выпил вчера?»
- «Закрыл ли норму белка?»
- «Как самочувствие? Уровень стресса?»

### 11.2 Еженедельный отчёт
- Сравнение веса и замеров с прошлой неделей
- Если прогресса нет 2 недели → P.A.R.K.E.R. автоматически пересчитывает КБЖУ
  (снижает калории Лере / добавляет Петру)
- Выполнение тренировок: сколько из запланированных закрыто

### 11.3 Фото-контроль (этап 4)
- Загрузка фото «До / После» в защищённое облако
- Визуальный анализ прогресса
- Хранение истории тела

### 11.4 Психологический блок
- При срыве: не штрафует, предлагает «Минимальный план»
- Поддерживающие сообщения, если человек долго не заходил
- «Ты пропустил 3 дня. Это нормально. Начнём с лёгкой тренировки сегодня?»

## 12. ТЕХНИЧЕСКАЯ ЛОГИКА (как работает внутри)

| Функция | Что делает P.A.R.K.E.R. |
|---|---|
| Адаптация питания | Меняет план на ходу, если пропущен приём пищи |
| Обход ограничений | Заменяет рыбу на креветки, учитывает аллергии |
| Тренерский надзор | Анализирует график и вставляет тренировку в свободное окно |
| Прогрессия | Помнит предыдущие нагрузки, предлагает рост |
| Психология | Поддерживает при срывах, не даёт «голодать» |
| Умный пересчёт | При стагнации — автоматически корректирует КБЖУ |
| Медицинский фильтр | 100% блокировка запрещённых упражнений по профилю |

## 10. ФОРМУЛЫ РАСЧЁТА КБЖУ

### BMR
**Кетч-МакАрдл** (приоритет, если указан % жира):
```
LBM = вес_кг × (1 − жир% / 100)
BMR = 370 + 21.6 × LBM
```

**Миффлин-Сан Жеор** (если % жира не указан):
```
Мужчины: BMR = 10×вес + 6.25×рост − 5×возраст + 5
Женщины: BMR = 10×вес + 6.25×рост − 5×возраст − 161
```

### TDEE = BMR × коэффициент активности
| График | Коэффициент |
|---|---|
| Стандартный (8ч) | 1.375 |
| Интенсивный (12ч) | 1.55 |
| Тяжёлый (16ч+) | 1.725 |
| Сменный | 1.55 |

### Макросы по цели
| Цель | Δ ккал | Белки | Жиры | Углеводы |
|---|---|---|---|---|
| Похудеть | −400 | 35% | 25% | 40% |
| Набор массы | +300 | 30% | 25% | 45% |
| Рекомпозиция | 0 | 35% | 30% | 35% |
| Поддержание | 0 | 25% | 30% | 45% |

## 11. БИЗНЕС-ПРАВИЛА (алгоритм)

| Условие | Действие системы |
|---|---|
| Сколиоз / проблемы со спиной | Запрет: становая тяга, приседания со штангой, гиперэкстензия с весом, наклоны с отягощением |
| Проблемы с коленями | Запрет: глубокие приседания с весом, выпады с отягощением, запрыгивания на тумбу |
| Гипертония | Запрет: тяжёлый жим над головой, натуживание (задержка дыхания) |
| График 16ч+ | Режим «Минимальный план»: 3 быстрых приёма, 5 мин растяжки, витамины |
| Нестабильный сон | Снижение интенсивности тренировки, акцент на восстановление |
| 12ч на ногах | Предложить верх тела или плавание вместо ног |
| Любой план | Обязательный медицинский дисклеймер |

## 12. РИСКИ И РЕШЕНИЯ

| # | Риск | Решение |
|---|---|---|
| 1 | **Медицинский:** травма при сколиозе | Алгоритмический запрет + дисклеймер в каждом плане |
| 2 | **Мотивация:** пользователь бросит из-за усталости | Режим «Минимальный план» — сохраняем привычку даже в самый тяжёлый день |
| 3 | **Технический:** учёт продуктов | Этап 1 — шаблоны; Этап 2 — FatSecret API |
| 4 | **Точность КБЖУ для экстремальных параметров** | Кетч-МакАрдл при наличии % жира; TDEE скорректирован под реальный график |
| 5 | **Потеря данных при рестарте** | MemoryStorage → Supabase на этапе 3 |
| 6 | **Зависимость от Claude API** | Шаблонный fallback на все случаи |
| 7 | **Масштабирование нагрузки** | Railway autoscale + кэш Redis на этапе 4+ |

-- Профиль пользователя
CREATE TABLE users (
    id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),

    username        text,

    gender          text,           -- male | female
